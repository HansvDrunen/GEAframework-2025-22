Eviden EDA and Modeling

This notebooks aims to classify company statements such as mission, vision, strategy, goals and core values using prelabeled dataset. First a basic eda is done to understand the data. Then two models are used: rule based baseline and TF-IDF and logistic regression classifier. Both models are evaluated using precision, recall and f1 score and the two models are then compared.

def clean_text(text):

def top_words_by_label_lang(df, label, lang_code, n=10):
    subset = df[(df["label"] == label) & (df["lang"] == lang_code)]
    all_words = [word for tokens in subset["tokens"] for word in tokens]
    word_counts = Counter(all_words)
    return word_counts.most_common(n)

For dutch the overall accuracy is even lover at about 14 percent. Some labels have a moderate recall, but most classes are predicted incorrectly. Weighted f1 is 0.15.

Overall it can be determined that a basic keyword classifier is nor effective in clasifing the statements and while it can be improved with more specific keyword that risk overfitting for this dataset.

def keyword_baseline_bilingual(text):

    if any(w in text for w in ["mission", "purpose", "serve", "build", "missie", "doel","belangrijke","rol"]):
        return "mission"

    elif any(w in text for w in ["vision", "future", "aspire", "aim", "visie", "toekomst","willen","nitiatieven"]):

Train TF-IDF + Logistic Regression

TF-IDF is used to turn text into numeric statistic to reflect the significance of a word. Using this to highlight keywords within the statement types. Logistic regression will learn the patterns of where these words are used to then seperate it into its types. We will use precision, recall and f1 score to measure the accuracy.

For english the overall accuracy is about 53 percent. Classes like core value and strategy are predicted well, with high recall for strategy. However, small classes like mission and vision are still poorly captured due to lower number of statements.

The accuracy for dutch is 46 percent, with better recall for goals. Classes like core value, mission, and vision perform poorly, because of limited training data.

Overall this is better than the basline and can be improved with more data as a starting point.

In[11]:
def train_logreg_tfidf(X_train, y_train, X_test, y_test):
    vectorizer = TfidfVectorizer(max_features=2000, ngram_range=(1,2))
    X_train_tfidf = vectorizer.fit_transform(X_train)
    X_test_tfidf = vectorizer.transform(X_test)

    clf = LogisticRegression(max_iter=1000)
    clf.fit(X_train_tfidf, y_train)
    y_pred = clf.predict(X_test_tfidf)

    report = classification_report(y_test, y_pred, output_dict=True, zero_division=0)
    return clf, vectorizer, report, y_pred

clf_en, vec_en, report_en, y_pred_en = train_logreg_tfidf(X_train_en, y_train_en, X_test_en, y_test_en)

clf_nl, vec_nl, report_nl, y_pred_nl = train_logreg_tfidf(X_train_nl, y_train_nl, X_test_nl, y_test_nl)

print("Logistic regression for english")
print(classification_report(y_test_en, y_pred_en, zero_division=0))

print("Logistic regression for dutch")

Results summary

The evaluation table shows the keyword-based baseline and a TF-IDF and logistic regression classifier. The baseline has low accuracy at about 16 percent in english and 14 percent in dutch. This is because it focuses only on keywords and not conext in the statements. In comparison, the TF-IDF and logistic regression classifier has about a 53 percent accuracy for english and 46 percent for dutch and also higher weighted f1 scores of 0.44 in english and and 0.37 in dutch.

Overall, the table confirms that using a model trained on TF-IDF features is more effective than a rule-based approach for classifying company statements.
